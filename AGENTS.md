# Agent Rules

## Purpose

This repository maintains current-state documentation and project history as part of normal development work.

## Required Files

- `README.md` describes the current project structure, active pages, branding, and run/build workflow.
- `HISTORY.md` records notable repository changes in chronological order.

## Update Rules

When editing this repository, always check whether the change affects any of the following:

- active overlay pages
- page names or entry points
- build inputs or output structure
- brand text or channel identity
- viewer instructions such as Discord or chat commands
- development or build workflow

If any of those change, update `README.md` and `HISTORY.md` in the same change set before finishing.

## Usage Rules

- Read `README.md` before making structural changes.
- Append new notable changes to `HISTORY.md` instead of rewriting prior entries.
- Do not mention removed pages in `README.md` as active pages.
- Keep documentation factual and synchronized with the current workspace state.
